We benchmarked a To-Do application backend built or deployed using 4 different platforms/frameworks i.e.
  • Rails app deployed on DigitalOcean
  • Rails app deployed using Heroku
  • App built and deployed using Hasura on DigitalOcean
  • App built using Firebase

The backend for each app powers the following features:
  • A user can view, add, update and delete their todos.
  • New user registration, login, and logout.

Infrastructure baseline

To baseline infrastructure across these different platforms/frameworks, the apps have been deployed either on VMs with exactly the same specs (4GB RAM, 2xCPU - DigitalOcean costing $40) or using equally priced plans on Firebase (Blaze plan) or Heroku ($25 compute + $9 database).
Hasura works with any infrastructure!
The Hasura platform is infrastructure agnostic and is installed on your own infrastructure (including your local machine). Hasura projects can be seamlessly migrated across providers.

Benchmarking throughput

To benchmark the different apps,

  • Multiple parallel threads were launched using jmeter in 1 second.
  • Each thread was made to perform authenticated CRUD operations sequentially.
  • This exercise was repeated 3 times and the average of these results has been considered.
Different number of threads were spawned to simulate low (10), standard (200) and heavy (1000) load conditions.

To see why Hasura performs so well, check out the subsequent sections!

Hasura BaaS components

For details on loc for other platforms/frameworks, please refer to their respective source code (linked at the bottom of the page)


Hasura provides built-in Data + Auth APIs.



For this exercise, the Data and Auth APIs were used directly as the backend. ~30 lines of code were written for the database schema and permissions.

Hasura BaaS components speed up your development time tremendously.

Data API: As soon as your create your database schema, you get a JSON based HTTP query API readily available on your data, with permissions and access control.


Hasura Data API performs so well because:

  • The Data API is written in Haskell and is optimized to exploit mutli-core CPUs to achieve high concurrency.
  • The Data API is also optimized to use PostgreSQL features which help in the performance. For example, it uses prepared statements and JSON aggregation queries of PostgreSQL.
  • The Data API does fast JSON serialization/deserialization.

Auth APIs: Readymade Auth APIs for authentication and user management.

Hasura PostgreSQL

Hasura comes pre-installed with a PostgreSQL instance for you.


Hasura deployment tools

Hasura can deploy code via:
  • Docker images
  • Git push your code

In this example, we used to git push to deploy our code (our UI service) by simply running:

$ git push hasura master

Hasura API Gateway

Hasura is bundled with an API gateway, which is an Nginx instance setup as a reverse proxy and is configured with SSL.



This gatway also handles centralized session management, using Redis to resolve auth-tokens into user/role HTTP headers for all upstream microservices.


Hasura also automatically creates free Grade A SSL certificates using Let's Encrypt.

Accessing your database

Hasura provides unrestricted access to its pre-installed PostgreSQL instance.

Writing custom server-side code

Hasura is a Microservices platform that allows you to deploy your custom code using Docker.

Owning your infrastructure

Hasura tooling is installed on your own infrastructure.
