Hasura on your infrastructure

Hasura platform can be installed on your own infrastructure.

For this comparison, we've installed Hasura on a DigitalOcean droplet.

Hasura data API

For this exercise, Hasura Data API was used as the backend.

Hasura Data API performs so well because:

  • The Data API is written in Haskell and is optimized to exploit mutli-core CPUs to achieve high concurrency.
  • The Data API is also optimized to use PostgreSQL features which help in the performance. For example, it uses prepared statements and JSON aggregation queries of PostgreSQL.
  • The Data API does fast JSON serialization/deserialization.

Hasura BaaS components

Hasura provides built-in Data + Auth APIs.

For this exercise, we used the Data and Auth APIs directly as our backend. We had to write ~30 lines of code for the database schema and permissions.

Hasura BaaS components speed up your development time tremendously:
  • As soon as your create your database schema, you get a HTTP query API readily available on your data, with permissions and access control.
  • Readymade Auth APIs for authentication and user management. You can use these and focus your efforts in bulding core parts of your app.

Hasura PostgreSQL

Hasura already installs and configures a PostgreSQL instance for you. On which the Hasura Data service runs.

You have access to this PostgreSQL instance directly as well.

Hasura deployment tools

Hasura has 2 easy ways to deploy custom code - in this case our UI service.
In this example, we used to git push to deploy our code by simply running:
$ git push hasura master

Hasura can deploy code via:
  • Docker images
  • Git push your code

Hasura API gateway

Hasura is bundled with an API gateway, which is an Nginx instance setup as a reverse proxy and is configured with SSL.

Hasura also creates free Grade A SSL certificates using Let's Encrypt.
